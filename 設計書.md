# マルチテナントとは　実験
## 目的
マルチテナント型SaaSにはいくつかの実現方法がある。
そのうちスキーマ分離と行単位分離について、セキュリティや利便性を比較したい。

## システムの概要
人事系システム。  
社員の一覧が見られる。

## システム構成
- Webサーバ2台(HTTPS化する)
- ロードバランサー
- DBサーバ(PostgresSQL)
- リバースプロキシサーバ
- ジョブキュー(Redis)
- ワーカサーバー

いずれもVirtualBox+Ubuntuの仮想サーバ

## API設計
- 自社の社員一覧取得
```
GET /employees
```

- 全社員エクスポート機能
```
GET /exports/employees
```
（バックグラウンドワーカーで処理）

## DB設計
1. company
id
companyName
（ダミーデータとしてA社、B社、C社を登録）

2. employee
id
companyId
employeeName
role 1:TENANTADMIN 2:USER
password
（各社3人ずつダミーデータを登録）

3. admin
id
name
password

RLSの場合は全テーブルをpublicスキーマに設置  
スキーマ分離の場合はcompanyとemployeeを各社スキーマに設置、adminを管理者用スキーマに設置
